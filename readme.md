## üìã JSON Configuration File Structure
#### datasets (Multiple datasets supported, indexed by 0, 1, 2, ...)
```bash

    #inside every dataset
    
    name:"sd_dataset"  #name of dataset
    raster_gt:"/mnt/wxq/ResDepth/DATA/new/SD1/SD_GT_Clip2.tif"  #DSM Ground Truth
    raster_in:"/mnt/wxq/ResDepth/DATA/new/SD1/SD_raw_Clip2.tif"  #initial DSM input of the datasetÔºàwaiting for refinementÔºâ
    path_image_list:"/mnt/wxq/ResDepth/DATA/new/SD1/imagelist_notCF.txt" #orthophoto- remote sensing image after ortho-rectifiedÔºàpay attention to the formatÔºâ
    path_pairlist_training:"/mnt/wxq/ResDepth/DATA/new/SD1/pairlist_stereo_notCF.txt" #orthophoto- remote sensing image after ortho-rectifiedÔºàpay attention to the formatÔºâ
    path_pairlist_validation:"/mnt/wxq/ResDepth/DATA/new/SD1/pairlist_stereo_notCF.txt" #same as above
    area_type:"train+val"  #choose train+val mode, must in training
    allocation_strategy:"5-crossval_vertical" 
    test_stripe:2
    n_training_samples:6000 #training samples amount for every dsm file
    crossval_training:true #cross validation usually set as true
    mask_ground_truth:"/mnt/wxq/ResDepth/DATA/new/SD1/loss_mask_SD1.tif" #path of the loss maskÔºåarea set as 1 represents the building area, which means ROI, involved in the loss calculation. If there is no mask set, then all the area takes part in the loss calculation.
```
